## 一、项目目标

构建一个基于认知架构理论的 AI Agent 系统。不基于现有 Agent 框架，从零实现 4.5 层认知模型。目标是验证：当 LLM 具备类似人类的分层记忆和自演化机制后，能否涌现出更高的智能行为。

**这是一个认知科学实验**，同时也是一个个人助手系统的长期基础。

---

## 二、4.5 层认知架构

### 总览

```
┌──────────────────────────────────────────────┐
│  L0.5  元驱动层 (Meta-Driver)                 │
│  手动配置 · 只读 · 不可被 Agent 修改           │
│  持续学习 · 反思失误 · 分解复杂任务             │
├──────────────────────────────────────────────┤
│  L1    哲学 / 行为准则层 (Philosophy)          │
│  自动演化 · 经 L0.5 反思验证后更新             │
│  行为准则集合 → 行动选择过滤器                 │
├──────────────────────────────────────────────┤
│  L2    柔性知识 / 领域经验 (Flexible Knowledge)│
│  概率性知识 · 带置信度和适用条件               │
│  类比检索 + 上下文激活 + 置信度衰减             │
├──────────────────────────────────────────────┤
│  L3    半静态能力 (Skills + Tools)             │
│  过程性记忆 · 编译后的可调用单元               │
│  由 L2 激活模式驱动创建，固化为 procedure       │
├──────────────────────────────────────────────┤
│  L4    静态知识库 (Static Knowledge Base)      │
│  声明性记忆 · 多索引 · 可无限扩展               │
│  领域知识库 + 通用知识库                        │
└──────────────────────────────────────────────┘
```

### 各层定义

| 层 | 类型 | 管理方式 | 内容示例 | 认知对应 |
|----|------|----------|----------|----------|
| L0.5 | 固定参数 | 手动配置，只读 | 持续学习、反思失误、分解复杂任务 | 认知架构的固定决策流程 |
| L1 | 可演化参数 | Agent 可修改，需 L0.5 验证 | "不编造信息"、"优先验证假设" | 元认知 / 行为准则 |
| L2 | 概率性知识 | 自动+手动，带置信度衰减 | "金融数据优先用权威源"、"可参考相邻领域" | 声明性记忆 (带激活值) |
| L3 | 过程性记忆 | 由 L2 驱动自动创建 | 数据分析 skill、代码审查 skill | 过程性记忆 (production rules) |
| L4 | 声明性记忆 | 写入后多索引存储 | 领域知识库 + 通用知识库 | 长时记忆存储 |

### 核心约束

- **上层可以读取/调用下层，下层不能直接修改上层**
- **L0.5 是唯一可以修改 L1 的入口**（通过反思验证机制）
- **L1 约束 L2 的激活条件**，L2 为 L3 的创建提供经验模板
- **L4 被所有层读取**，只被 L0.5 和 L2 写入

## 三、技术路线

### 实现方式：Approach B — 认知架构从零构建

不基于 LangChain、AutoGPT 等现有 Agent 框架。参考 ACT-R、Soar 等经典认知架构的设计原则，每层独立实现状态管理和更新机制。

**参考理论来源**:
- ACT-R: L2(declarative chunks) + L3(production rules) 的分离
- Constitutional AI: L0.5 固定宪法的思想来源
- CoALA: 统一描述 LLM Agent 认知架构的理论框架
- MemGPT: L4 分层存储的工程参考

### 渐进验证路径

```
TextWorld 微型验证 → 复杂 TextWorld 关卡 → (可选) NetHack → 代码生成 → 个人助手
```

| 阶段 | 环境 | 验证目标 |
|------|------|----------|
| 1 | TextWorld 微型世界 | 最小闭环验证：3-4 房间，找钥匙→开门→拿宝藏 |
| 2 | 复杂 TextWorld 关卡 | 渐进复杂度下各层学习机制的稳定性 |
| 3 | NetHack (可选) | 在更开放环境中验证架构的通用性 |
| 4 | 代码生成 | 验证知识从游戏到现实领域的迁移性 |
| 5 | 个人助手 | 最终目标：开放的日常任务处理 |

## 四、Phase 1 — 可评估任务导向框架

### 设计原理

第一阶段限制在**有客观判定标准的任务**上。用户输入被解析为一个有明确目标且可评估的"任务"，Agent 完成后自动判定成功/失败。其他类型的输入暂不处理。

核心机制：

```
用户需求 → 任务定义 (可评估规格)
  → Agent 执行 (4.5 层认知处理)
  → 自动评估 (客观判定)
  → 分层归因 (成功/失败归因到具体层)
  → 定向更新 (更新对应层的状态)
  → 循环
```

### 为什么不用纯强化学习

传统 RL: state → action → scalar reward → single weight update

本系统: task context → 4.5 层认知处理 → evaluable outcome → **多层结构化反思**

区别在于：失败信号不是均匀地惩罚所有参数，而是进行分层归因——失败是因为知识缺失(L4)、经验不足(L2)、行为策略不对(L1)、还是任务分解有误(L0.5)？

### 评估设计原则

- **非二元**：不只是通过/失败。区分语法错误、逻辑错误、部分正确等不同失败类型
- **过程化奖励**：任务执行过程中间步骤也有评估信号
- **不同失败类型 → 不同归因层级**

### 冷启动策略

系统初始没有 L2 领域经验、L3 skill 也几乎为空。为避免早期反馈信号稀疏，注入初始种子：
- 人工标注的种子 L2 经验
- 基础 L3 技能
- 初始 L4 知识条目（规则说明、物品关系等）

## 五、为什么选择 TextWorld 作为起点

### 游戏环境 vs 其他领域

| 特征 | TextWorld | SQL/数据分析 | 代码生成 |
|------|-----------|-------------|----------|
| 评估基础设施 | 环境自带 | 需要搭建 | 需要测试框架 |
| 反馈延迟 | 即时 | 即时 | 秒级 |
| 过程化信号 | 每一步 | 有限 | 编译/测试结果 |
| 安全沙箱 | 天然 | 需要 | 需要 |
| 可并行化 | 是 | 是 | 是 |
| 可复现 | 是 | 是 | 部分 |
| 渐进复杂度 | 自定义关卡 | 有限 | 单函数→项目 |
| 知识可迁移性 | 低 | 中 | 高 |

游戏环境作为架构验证实验场的核心优势：**评估和反馈是环境自带的，不需要设计**。每一步动作环境都返回新状态，不需要等编译、不需要人工标注。正在验证的是架构机制本身，不是特定领域的知识积累。

### 架构层在游戏中的映射

```
游戏场景                      →  架构层
─────────────────────────────────────────
卡关，反复死在同一处            →  L0.5 触发反思
"这个boss要先破防再输出"       →  L2 领域经验形成
"打boss的连招套路"             →  L3 半静态技能固化
地图、道具属性、怪物图鉴       →  L4 静态知识库
"打不过先探索找装备，不要硬莽"  →  L1 行为准则更新
```

## 六、关键设计挑战

### 待解决问题

1. **L1 自动调整的验证边界**：Agent 可以修改 L1，但需要类似"回归测试"的行为校验机制防止退化

2. **L2 置信度衰减模型**：领域经验有半衰期，需要定义衰减曲线——基于时间？基于使用频率？基于成功/失败反馈？

3. **L0.5 反思触发条件**：定时触发 vs 事件驱动 vs 混合。太频繁浪费计算，太少错过关键信号。需要定义触发条件（任务失败、不确定、进入新领域）

4. **分层归因精度**：一个任务失败可能同时涉及多层（L4 知识缺失 + L1 策略错误），如何精确归因？
   - 排除法：如果换一个更好的 L3 skill 能解决问题 → 归因到 L3
   - 如果补充 L4 知识能解决 → 归因到 L4

5. **知识迁移**：从 TextWorld 切到代码生成时保留什么、重置什么？
   - 保留：L0.5(通用)、L1(经过验证的通用行为准则)
   - 重置：L2、L3、L4 中领域特定的内容
