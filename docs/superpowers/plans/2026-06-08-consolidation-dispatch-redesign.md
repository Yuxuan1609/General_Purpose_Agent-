# Consolidation Task Dispatch Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the monolithic `meta` blob with per-layer `state["lX_task"]` fields, fix L3 dispatch, give each layer targeted consolidation context with judgment criteria, and add quality fields (`usefulness`/`misleading`/`comment`) to Rule/KnowledgeCard/SkillMeta.

**Architecture:** `build_consolidation_task()` produces slim `meta` (header only) + `state["l1_task"]` (criteria text) + `state["l2_task"]` / `state["l3_task"]` (target_domains dict). L2/L3 managers fetch their own cards/skills by domain and format display lines with DD3 fields. L2 stage1 detects consolidation and forces `call_l3=true`. `_filter_meta_for_layer()` removed.

**Tech Stack:** Python 3.13, existing `core/env/learning_env.py`, `core/layers/l*/manager.py`, `core/layers/base.py`, `core/flexible_knowledge.py`, `core/philosophy.py`, `core/skill_layer.py`

## Design Decisions (confirmed 2026-06-08)

### DD1: Dual trigger — capacity + modification count

- **Trigger A (capacity)**: per-domain count > soft/hard limit treated as "capacity overflow"
- **Trigger B (modification)**: per-domain change counter >= 5 since last consolidation treated as "routine maintenance"
- Counter stored in `learning_stats.json` under `_consolidation.<domain>.mod_count`
- Counter reset to 0 after that domain is consolidated
- Only triggered domains appear in `state["lX_task"]`; untriggered domains are not listed

### DD2: Quality fields — usefulness / misleading / comment

Three new fields added to `Rule`, `KnowledgeCard`, and `SkillMeta` dataclasses:

| Field | Type | Default | Updated by |
|-------|------|---------|-----------|
| `usefulness` | `int` | `0` | reflect/consolidate LLM via modify tools |
| `misleading` | `int` | `0` | reflect/consolidate LLM via modify tools |
| `comment` | `str` | `""` | reflect/consolidate LLM via modify tools (<= 100 chars) |

- Updated via `modify_l1_rule` / `modify_l2_card` / `modify_l3_skill` (optional parameters, not in `required`)
- Trigger: tasks involving reflection/feedback (path 1: LearningEnv reflect, path 2: consolidation when appropriate)
- LLM only updates entries it has a clear opinion on; not required to comment on every entry

### DD3: Consolidation card display format

Per-card line (generated by L2Manager, not by env):
```
- [card_id] conf=0.8 used=3 last=2026-06-05 useful=+3 mislead=0 | content[:120]
  comment: useful reference for new agents, covers pre-flop K strategy well
```

- `conf` from card object (seed value)
- `used` / `last` from stats file (live quality signals)
- `usefulness` / `misleading` / `comment` from card object (reflect feedback)
- `domain` omitted from per-line (already in section header)
- `activation` / `succ` / `fail` / `decay` omitted

Per-skill line (generated by L3Manager):
```
- [skill_name] used=3 last=2026-06-05 useful=+1 mislead=0 | description[:120]
  comment: well-structured, covers all pre-flop scenarios
```

### DD4: Env injection + layer self-retrieval

- `state["l1_task"]` = judgment criteria text only (no rule listing; L1 system prompt already has rules)
- `state["l2_task"]` = `{"target_domains": [...], "criteria": "...", "triggers": {domain: "capacity"|"maintenance"}}`
- `state["l3_task"]` = `{"target_domains": [...], "criteria": "...", "triggers": {domain: "capacity"|"maintenance"}}`
- L2Manager fetches cards from `self._knowledge` filtered by `target_domains`, formats per DD3
- L3Manager fetches skills from `self._skill_layer` filtered by `target_domains`, formats per DD3
- `session["domains_hint"]` = `["learning/compile"] + target_domains`
- L1 is special: no domain involved, no data to fetch; `state["l1_task"]` is just criteria text

```python
def build_consolidation_task(self) -> TaskObservation | None:
    l2 = self._knowledge.get("l2")
    l3 = self._knowledge.get("l3")
    l1_rules = self._knowledge.get("l1")
    needs_l2 = l2 and len(l2.cards) > self._l2_limit
    needs_l3 = l3 and len(l3.list_all()) > self._l3_limit

    # ── Build triggered domain lists ──
    cons_state = self._stats.get("_consolidation", {})
    spec = self._consolidation_spec

    l2_limits = spec.get("l2", {}).get("limits", {}) if spec else {}
    l2_soft = l2_limits.get("per_domain_soft", l2_limits.get("soft", 25)) if l2_limits else 25
    l2_hard = l2_limits.get("per_domain_hard", l2_limits.get("hard", 30)) if l2_limits else 30

    l3_limits = spec.get("l3", {}).get("limits", {}) if spec else {}
    l3_soft = l3_limits.get("per_domain_soft", l3_limits.get("soft", 15)) if l3_limits else 15
    l3_hard = l3_limits.get("per_domain_hard", l3_limits.get("hard", 20)) if l3_limits else 20

    l2_triggers: dict[str, str] = {}  # domain -> "capacity" | "maintenance"
    l3_triggers: dict[str, str] = {}

    from collections import Counter
    all_domains: set[str] = set()

    if needs_l2 and l2:
        domain_counts: Counter[str] = Counter()
        for c in l2.cards:
            domain_counts[c.domain.path] += 1
            all_domains.add(c.domain.path)
        for domain, count in domain_counts.items():
            mods = cons_state.get(domain, {}).get("mod_count", 0)
            if count > l2_soft:
                l2_triggers[domain] = "capacity"
            elif mods >= 5:
                l2_triggers[domain] = "maintenance"

    if needs_l3 and l3:
        skill_domain_counts: Counter[str] = Counter()
        for s in l3.list_all():
            skill_domain_counts[s.domain.path] += 1
            all_domains.add(s.domain.path)
        for domain, count in skill_domain_counts.items():
            mods = cons_state.get(domain, {}).get("mod_count", 0)
            if count > l3_soft:
                l3_triggers[domain] = "capacity"
            elif mods >= 5:
                l3_triggers[domain] = "maintenance"

    # If no triggers at all, nothing to do
    if not l2_triggers and not l3_triggers:
        return None

    level = self.get_consolidation_level()
    level_info = spec.get("consolidation_levels", {}).get(level, {}) if spec else {}

    # ── meta: slim header ──
    meta_lines = [
        "## Knowledge Consolidation Task",
        f"**Level: {level} {level_info.get('label', '')}**",
        "",
    ]
    if l2_triggers:
        meta_lines.append(f"L2 Knowledge Cards to review: {len(l2_triggers)} domain(s) triggered")
    if l3_triggers:
        meta_lines.append(f"L3 Skills to review: {len(l3_triggers)} domain(s) triggered")
    meta_lines.append("")
    meta_lines.append("Each layer has its own task in state. Only modify your own layer's data.")
    meta = "\n".join(meta_lines)

    # ── l1_task: criteria text only ──
    l1_task = (
        "## L1 Task\n\n"
        "Review your behavior rules in the system prompt against the criteria below.\n\n"
        "### Judgment Criteria\n"
        "- **deprecate**: duplicate rules (semantic similarity > 80%), domain-specific content "
        "(should belong to L2), vague rules (< 20 chars with no specific directive), "
        "violates cross-domain methodology principle\n"
        "- **modify**: merge multiple rules into one while preserving complete semantics; "
        "also use modify to update usefulness/misleading/comment for rules "
        "that were helpful or misleading during reflection\n"
        "- **create**: only for new cross-domain methodology not achievable via modify\n\n"
        "Use tools: deprecate_l1_rule / create_l1_rule / modify_l1_rule"
    )

    # ── l2_task: target_domains dict ──
    l2_task = json.dumps({
        "criteria": (
            "### Judgment Criteria\n"
            "- **deprecate**: used=0 + conf<0.2, placeholder/experimental content, "
            "content semantically > 80% similar to another card (keep higher-quality one)\n"
            "- **modify**: merge >= 2 similar cards in same domain into one refined card; "
            "also use modify to update usefulness/misleading/comment for cards "
            "that were helpful or misleading during reflection\n"
            "- **create**: cross-domain generalization (multiple similar cards -> one higher-level card)"
        ),
        "target_domains": list(l2_triggers.keys()),
        "triggers": l2_triggers,
    }, ensure_ascii=False)

    # ── l3_task: target_domains dict ──
    l3_task = json.dumps({
        "criteria": (
            "### Judgment Criteria\n"
            "- **deprecate**: never matched/used (use_count=0), vague content, "
            "functionally overlaps with another skill\n"
            "- **modify**: add missing process steps, update outdated instructions; "
            "also use modify to update usefulness/misleading/comment\n"
            "- **create**: compile >= 3 high-activation same-domain cards into SKILL.md "
            "(write full YAML frontmatter + Markdown procedure)"
        ),
        "target_domains": list(l3_triggers.keys()),
        "triggers": l3_triggers,
    }, ensure_ascii=False)

    hint_domains = ["learning/compile"] + sorted(all_domains)

    return TaskObservation(
        meta=meta,
        state={
            "current": "Consolidation task - see per-layer tasks below",
            "history": "",
            "l1_output_format": _L1_OUTPUT,
            "l2_output_format": _L2_OUTPUT,
            "l3_output_format": _L3_OUTPUT,
            "l1_task": l1_task,
            "l2_task": l2_task,
            "l3_task": l3_task,
            "feedback": self._shared_feedback,
            "l1_feedback": self._layer_feedback.get("l1", ""),
            "l2_feedback": self._layer_feedback.get("l2", ""),
            "l3_feedback": self._layer_feedback.get("l3", ""),
        },
        session={
            "domain": "learning/compile",
            "domains_hint": list(dict.fromkeys(hint_domains)),
            "id": f"consolidation_{self._step_count}",
            "step_index": 0,
            "enable_learning": False,
        },
    )
```

```python
def _format_consolidation_cards(self, domains: list[str], stats: dict) -> str:
    """Build per-domain card listing with DD3 fields."""
    lines = []
    for domain_path in sorted(domains):
        cards = [c for c in self._knowledge.cards if c.domain.path == domain_path]
        if not cards:
            continue
        lines.append(f"### {domain_path} ({len(cards)} cards)")
        for c in cards:
            st = stats.get("l2", {}).get(c.id, {})
            comment_line = f"\n  comment: {c.comment}" if c.comment else ""
            lines.append(
                f"- [{c.id}] conf={c.confidence:.1f} used={st.get('use_count', 0)} "
                f"last={st.get('last_used', '-')[:10]} useful=+{c.usefulness} "
                f"mislead={c.misleading} | {c.content[:120]}{comment_line}"
            )
        lines.append("")
    return "\n".join(lines)
```

def _format_consolidation_skills(self, domains: list[str], stats: dict) -> str:
    lines = []
    for domain_path in sorted(domains):
        skills = [s for s in self._skill_layer.list_all() if s.domain.path == domain_path]
        if not skills:
            continue
        lines.append(f"### {domain_path} ({len(skills)} skills)")
        for s in skills:
            st = stats.get("l3", {}).get(s.name, {})
            comment_line = f"\n  comment: {s.comment}" if s.comment else ""
            lines.append(
                f"- [{s.name}] used={st.get('use_count', 0)} "
                f"last={st.get('last_used', '-')[:10]} useful=+{s.usefulness} "
                f"mislead={s.misleading} | {s.description[:120]}{comment_line}"
            )
        lines.append("")
    return "\n".join(lines)
