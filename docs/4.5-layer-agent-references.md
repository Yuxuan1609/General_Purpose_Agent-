## 一、认知架构理论基础 (L0.5–L1 核心参考)

### ⭐⭐⭐ CoALA: Cognitive Architectures for Language Agents
- **作者**: Sumers et al. (Princeton / Microsoft, 2023)
- **链接**: https://arxiv.org/abs/2309.02427
- **摘要**: 提出将 LLM Agent 映射到经典认知架构的统一框架。定义 Agent 由工作记忆(活跃上下文) + 长时记忆(语义/情景/过程性 三个子类) + 决策流程 + 学习流程组成。明确将 Voyager、AutoGPT、Generative Agents 等现有工作纳入同一理论语言。**你的 4.5 层模型可以直接用 CoALA 的术语体系来精确描述。**
- **对应层**: L0.5 (决策流程) / L1 (学习流程) / L2-L4 (三种长时记忆)

### ⭐⭐ ACT-R 认知架构
- **作者**: John R. Anderson et al. (CMU, 持续演进)
- **链接**: http://act-r.psy.cmu.edu
- **摘要**: 基于人类认知的计算模型。核心分离: 声明性记忆(chunks, 带激活值衰减和检索概率) + 过程性记忆(production rules, if-then 条件-动作对)。知识通过激活扩散(spreading activation)被检索，通过 chunking 机制从声明性知识编译为过程性技能。**你的 L2(柔性知识) → L3(半静态技能) 的演化路径直接对应 ACT-R 的 chunking 理论。**
- **对应层**: L2 (声明性记忆 chunks) / L3 (过程性记忆 production rules) / L0.5 (activation-based retrieval 作为元驱动)

### ⭐⭐ Soar 认知架构
- **作者**: Laird, Newell, Rosenbloom et al. (University of Michigan, 1980s–至今)
- **链接**: https://soar.eecs.umich.edu
- **摘要**: 最古老且仍在活跃开发的认知架构之一。以 production rules 为核心执行单元，具有明确的 chunking 学习机制(将问题解决路径自动编译为新规则)。现代 Soar 研究探索集成 LLM 用于自然语言理解和知识获取。**其 chunking 机制是 L3 skill 自动创建的直接理论支撑。**
- **对应层**: L1 (operator 选择 = 行为准则) / L3 (chunking = skill 创建) / L0.5 (impasse-driven 反思)

### ⭐ OpenCog / Hyperon
- **作者**: Ben Goertzel et al. (OpenCog Foundation)
- **链接**: https://github.com/opencog
- **摘要**: 面向 AGI 的开源认知架构。核心是 AtomSpace——一个加权超图知识表示系统，支持多种认知过程(推理、注意力分配、学习)在同一知识库上并行运行。Hyperon 项目尝试将 LLM 集成进认知架构作为组件。**如果你的 L4 需要超越向量检索，AtomSpace 的加权超图是值得参考的设计。**
- **对应层**: L4 (AtomSpace 超图存储) / L2 (注意力分配 = 知识激活)

## 二、反思与自演化机制 (L0.5–L1)

### ⭐⭐⭐ Reflexion: Language Agents with Verbal Reinforcement Learning
- **作者**: Shinn et al. (2023)
- **链接**: https://arxiv.org/abs/2303.11366
- **摘要**: Agent 在任务失败后生成**文本形式的自我反思**，存储在情景记忆缓冲区中，作为后续尝试的额外上下文。实现了无需权重更新的"语言梯度下降"——Agent 从过去的错误中学习。评估了三种反馈类型: 二元成败、LLM 自我批评、外部验证信号。在 HotPotQA、HumanEval、ALFWorld 上验证。**这是 L1 自动调整的核心机制参考——Agent 通过文本反思更新自己的行为准则。**
- **对应层**: L0.5 (触发反思) / L1 (反思结果更新行为准则)

### ⭐⭐⭐ Generative Agents: Interactive Simulacra of Human Behavior
- **作者**: Park et al. (Stanford, 2023)
- **链接**: https://arxiv.org/abs/2304.03442
- **摘要**: 25 个 LLM Agent 在虚拟小镇中的生活模拟。核心架构: 记忆流(memory stream, 按时间记录所有事件) → 定期反思(synthesis, 将相关记忆合成高层抽象) → 递归规划(recursive task decomposition)。Agent 展现出涌现的社会行为(组织派对、传播信息)。**记忆→反思→规划的循环是你的 L4→L2→L1 数据流的直接参照。**
- **对应层**: L4 (记忆流) / L2 (反思合成) / L1 (规划行为) / L0.5 (反思触发周期)

### ⭐⭐ Self-Refine: Iterative Refinement with Self-Feedback
- **作者**: Madaan et al. (2023)
- **链接**: https://arxiv.org/abs/2303.17651
- **摘要**: 单任务内的迭代自我改进循环: 生成 → 自我批评 → 精炼。无需额外训练数据。在对话、代码生成、数学推理等多个领域验证有效。**比 Reflexion 更轻量——适合 L1 在线行为调整的场景。**
- **对应层**: L1 (在线行为调整)

### ⭐ Constitutional AI: Harmlessness from AI Feedback
- **作者**: Bai et al. (Anthropic, 2022)
- **链接**: https://arxiv.org/abs/2212.08073
- **摘要**: AI 系统遵循人类编写的"宪法"原则集。两阶段训练: (1) 监督阶段: LLM 生成回答后参照宪法自我批评并修订；(2) RL 阶段: 宪法模型评估回答偏好，产生 RLAIF 的训练数据。宪法本身是人类编写的、固定的。**你的 L0.5 设计——手动配置、Agent 不可修改——直接对应 Constitutional AI 的"固定宪法"思想。**
- **对应层**: L0.5 (人类编写的不可变宪法)

### ⭐ Tree of Thoughts
- **作者**: Yao et al. (2023)
- **链接**: https://arxiv.org/abs/2305.10601
- **摘要**: 扩展思维链为分支探索 + 自我评估 + 回溯。让 LLM 在推理过程中进行"深思熟虑的反思"——考虑多条路径并在它们之间做出选择。**为 L0.5 的"分解复杂任务"提供了具体的搜索/规划机制。**
- **对应层**: L0.5 (任务分解与搜索策略)

## 三、技能自动创建 (L3)

### ⭐⭐⭐ Voyager: An Open-Ended Embodied Agent with Large Language Models
- **作者**: Wang et al. (NVIDIA / Caltech / Stanford, 2023)
- **链接**: https://arxiv.org/abs/2305.16291
- **摘要**: Minecraft 中的 LLM Agent，无需人类干预即可探索世界、学习技能、做出发现。三个核心组件: (1) 自动课程——开放式探索驱动；(2) **技能库**——Agent 自己编写、复用、逐步优化的可执行 JavaScript 代码，通过 embedding 索引检索；(3) 迭代提示机制——结合环境反馈、执行错误、自我验证来改进代码。**这是 L3 自动 skill 创建最接近的现有实现。**
- **对应层**: L3 (可复用技能库) / L0.5 (自动课程 = 探索驱动)

### ⭐⭐ CREATOR: Tool Creation for LLM Agents
- **作者**: Qian et al. (2024)
- **链接**: https://arxiv.org/abs/2305.14339
- **摘要**: 当现有工具不够用时，Agent 自行创建新工具(Python 函数)来解决任务。工具保留在可复用库中。**直接对应 L3 的半静态能力创建流程——从 L2 的经验模式识别到 L3 的工具固化。**
- **对应层**: L3 (工具/技能创建与复用) / L2 (识别何时需要新工具)

### ⭐⭐ Toolformer: Language Models Can Teach Themselves to Use Tools
- **作者**: Schick et al. (Meta, 2023)
- **链接**: https://arxiv.org/abs/2302.04761
- **摘要**: 通过自监督方式在预训练语料中插入 API 调用，让 LLM 学会自主决定何时及如何使用外部工具(计算器、搜索、翻译)。**L3 的工具调用能力的训练方法论参考。**
- **对应层**: L3 (工具调用决策)

### ⭐ DSPy: Declarative Prompt Programming
- **作者**: Khattab et al. (Stanford NLP, 2023)
- **链接**: https://github.com/stanfordnlp/dspy
- **摘要**: 声明式指定 LLM 流水线，编译器自动搜索最佳 prompt 结构、few-shot 示例和微调策略来优化端到端性能。**将"声明式意图"编译为"可执行流水线"——这种编译思路对 L3 skill 固化有参考价值。**
- **对应层**: L3 (从经验到可执行 skill 的"编译"过程)

## 四、层次化记忆与知识管理 (L4)

### ⭐⭐⭐ MemGPT: Towards LLMs as Operating Systems
- **作者**: Packer et al. (UC Berkeley, 2023)
- **链接**: https://arxiv.org/abs/2310.08560
- **摘要**: 将 LLM 的上下文窗口视为**虚拟内存**。实现 OS 风格的分层记忆: 主上下文(类比 RAM，有限大小，活跃页面) + 外部/归档存储(类比磁盘，大容量持久存储)。Agent 通过基于 LLM 的中断机制自主管理数据在主存和外存之间的移动。**Letta** (https://github.com/letta-ai/letta) 是商业化框架。**L4 多索引知识库的"无限扩展"目标直接对应 MemGPT 的外存管理思想。**
- **对应层**: L4 (分层存储架构) / L0.5 (自主内存管理中断)

### ⭐⭐⭐ HippoRAG: Neurobiologically Inspired RAG
- **作者**: Gutierrez et al. (OSU / Stanford, 2024)
- **链接**: https://arxiv.org/abs/2405.14831
- **摘要**: 受海马体记忆索引理论启发。使用 Personalized PageRank 在知识图谱上进行多跳关联检索，模拟大脑的"模式补全"(pattern completion)能力。**你的"多次索引"设计和 L2 的类比检索可以参考 HippoRAG 的关联检索机制。**
- **对应层**: L4 (多索引关联检索) / L2 (类比/跨域激活)

### ⭐⭐ GraphRAG: Graph-Based Retrieval Augmented Generation
- **作者**: Microsoft (2024)
- **链接**: https://arxiv.org/abs/2404.16130
- **摘要**: 从文本构建知识图谱，通过社区检测和摘要生成实现多层次检索——结合图结构和 LLM 摘要的优势。**在你的 L4 中结合知识图谱索引和语义索引的具体方案参考。**
- **对应层**: L4 (知识图谱 + 社区摘要索引)

### ⭐⭐ RAPTOR: Recursive Abstractive Processing for Tree-Organized Retrieval
- **作者**: Sarthi et al. (2024)
- **链接**: https://arxiv.org/abs/2401.18059
- **摘要**: 通过递归聚类和抽象摘要构建文档的层次化树状结构，实现多粒度检索——从具体细节到高层摘要。**L4 知识库的多层级组织方案，特别适合领域知识的层次化归纳。**
- **对应层**: L4 (层次化索引) / L2 (高层摘要 = 领域经验)

### ⭐⭐ LlamaIndex
- **作者**: Jerry Liu et al.
- **链接**: https://github.com/run-llama/llama_index
- **摘要**: LLM 数据框架，提供多种索引策略: 向量索引、树索引、关键词索引、知识图谱索引、列表索引，支持组合使用。是当前"多索引 RAG"最成熟的开源实现。**实现 L4 时可以参考的工程方案。**
- **对应层**: L4 (多索引策略参考实现)

### ⭐ HyDE: Hypothetical Document Embeddings
- **作者**: Gao et al. (2023)
- **链接**: https://arxiv.org/abs/2212.10496
- **摘要**: 从查询生成假设文档，再用假设文档的 embedding 去检索——在 embedding 空间中创造查询侧的"索引"。零样本情况下显著优于直接查询 embedding。**L4 检索策略的一个补充技巧。**
- **对应层**: L4 (查询增强检索)

## 五、中文生态相关项目

### ⭐⭐ XAgent (清华 / OpenBMB)
- **链接**: https://github.com/OpenBMB/XAgent
- **摘要**: 自主 Agent，自动将复杂任务分解为子任务、动态工具调度、自我纠错。**你的 L0.5 "分解复杂任务" + L3 "工具调度"的中文生态参照。**

### ⭐ Qwen-Agent (阿里 / 通义)
- **链接**: https://github.com/QwenLM/Qwen-Agent
- **摘要**: 基于 Qwen 系列模型的 Agent 框架，支持长上下文推理、工具调用、代码解释器、有状态对话记忆。

### ⭐ ModelScope-Agent (阿里)
- **链接**: https://github.com/modelscope/modelscope-agent
- **摘要**: 阿里开源 Agent 框架，支持工具定制、多轮规划、代码执行、可扩展工具库。

### ⭐ AgentVerse (清华)
- **链接**: https://github.com/OpenBMB/AgentVerse
- **摘要**: 多 Agent 模拟框架，可定制环境与行为，Agent 可从交互中学习。
